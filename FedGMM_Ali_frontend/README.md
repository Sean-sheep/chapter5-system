## 系统架构方案（待确认）— 必读

**位置**：`06_系统架构方案（待确认）/`

### 核心约定

1. **双线逻辑**
   - **训练过程展示**：数据来自**训练过程日志**。用于证明第三章（通信压缩、γ）、第四章（τ* 动态、可选隐私）的创新。
   - **用户推荐使用**：数据来自**已训练模型推理**。用于证明“个性化推荐更精准”（按城市推荐消费品、多城市对比）。

2. **数据来源**
   - 证明创新点所需的参数与曲线（通信量、压缩比例、τ*、γ、隐私等）→ **全部来自训练日志**。
   - 个性化推荐结果（消费品列表等）→ **全部来自已训练模型的推理**。
   - 本系统不执行训练、不修改训练代码。

3. **页面划分**
   - **训练过程展示区**：通信与压缩效果、自适应迭代 τ* 轨迹、个性化权重 γ 分析、（可选）隐私与效用。
   - **用户推荐使用区**：按城市推荐（消费品）、多城市推荐对比。

4. **实现顺序**
   - 请您先阅读并确认 `系统整体逻辑与架构设计.md` 与 `页面与创新点对应表.md` 中的方案与确认清单。
   - **确认本方案后，再进入代码实现阶段。**

## 系统目标（简要）

- 展示**第三章**创新：通信开销低、多子模型 + 客户端混合权重 γ、个性化推荐更精准。
- 展示**第四章**创新：聚合时本地迭代次数 τ* 动态变化、可选差分隐私与效用。
- 技术栈建议：Python + Streamlit，数据存本地文件（训练日志 + 模型文件），不涉及数据库与训练计算。
