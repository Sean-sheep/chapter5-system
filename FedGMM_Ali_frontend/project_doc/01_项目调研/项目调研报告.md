# 项目调研报告：个性化联邦学习推荐系统开源项目

## 调研目标

寻找满足以下条件的GitHub开源项目，用于构建"城市旅游消费个性化推荐系统"展示平台：

### 必须满足条件
- ✅ 使用标准推荐数据集（MovieLens / Amazon / Yelp 等）
- ✅ 实现个性化联邦学习
- ✅ 包含通信压缩（Top-k / 稀疏上传 / 梯度裁剪等）
- ✅ 支持可配置的本地迭代次数（最好支持自适应策略）
- ✅ 代码结构清晰，可导出训练日志

### 优先考虑项目
- FedRec 系列
- FedFomo
- FedAMP
- pFedMe
- FedGMM
- SCAFFOLD + personalization 改进版本

---

## 推荐项目一：FedRec（Federated Recommendation）

### 项目信息
- **GitHub仓库**: FedRec 系列项目（多个变体）
- **主要特点**: 专门针对推荐系统的联邦学习框架
- **数据集支持**: MovieLens, Amazon, Yelp

### 满足条件分析

| 条件 | 满足情况 | 说明 |
|------|---------|------|
| 标准推荐数据集 | ✅ | 支持MovieLens、Amazon等主流数据集 |
| 个性化联邦学习 | ✅ | 核心功能，支持客户端个性化模型 |
| 通信压缩 | ⚠️ 部分 | 部分FedRec变体支持Top-k压缩，需确认 |
| 自适应本地迭代 | ⚠️ 需改造 | 通常支持固定本地迭代次数，自适应需自行实现 |
| 代码结构清晰 | ✅ | 推荐系统专用，结构相对清晰 |
| 训练日志导出 | ✅ | 通常支持日志记录 |

### 改造成展示系统的可行性
- **优势**: 专门针对推荐系统，业务场景匹配度高；代码结构清晰；推荐结果展示直观
- **劣势**: 通信压缩功能可能不完整；自适应本地迭代需要自行实现或改造
- **改造难度**: ⭐⭐⭐ (中等)
- **推荐指数**: ⭐⭐⭐⭐ (4/5)

---

## 推荐项目二：FedFomo（Federated Follow-the-Moving-Leader）

### 项目信息
- **主要特点**: 个性化联邦学习，客户端选择多个全局模型进行聚合
- **数据集支持**: 通常支持图像分类，推荐系统需适配

### 满足条件分析

| 条件 | 满足情况 | 说明 |
|------|---------|------|
| 标准推荐数据集 | ⚠️ 需适配 | 主要面向图像分类，推荐系统需改造 |
| 个性化联邦学习 | ✅ | 核心优势，客户端个性化权重机制 |
| 通信压缩 | ⚠️ 需补充 | 通常不包含压缩机制 |
| 自适应本地迭代 | ⚠️ 需改造 | 支持固定迭代次数 |
| 代码结构清晰 | ✅ | 算法逻辑清晰 |
| **推荐指数**: ⭐⭐⭐ (3/5)

## 推荐项目三：FedGMM（Federated Gaussian Mixture Model）

### 项目信息
- **主要特点**: 基于高斯混合模型的联邦学习，天然支持个性化
- **数据集支持**: 需确认是否有推荐系统实现

### 满足条件分析

| 条件 | 满足情况 | 说明 |
|------|---------|------|
| 标准推荐数据集 | ⚠️ 需确认 | 需确认是否有推荐系统实现 |
| 个性化联邦学习 | ✅ | 混合模型天然支持个性化 |
| 通信压缩 | ⚠️ 需补充 | 通常不包含压缩机制 |
| 自适应本地迭代 | ⚠️ 需改造 | 支持固定迭代次数 |
| 代码结构清晰 | ✅ | 数学理论清晰 |
| **推荐指数**: ⭐⭐⭐⭐ (4/5)

- 业务场景匹配度高；数据集支持完整；改造工作量适中；展示效果直观

## 调研结论

1. FedRec 系列最适合作为基础项目。
2. 通信压缩若不完整，可从其他项目借鉴或自行实现 Top-k + 残差反馈。
3. 自适应本地迭代次数 τ* 多数需自行实现。
4. 展示系统以读取训练结果与可视化为核心，改造难度可控。
